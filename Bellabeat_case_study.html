Bellabeat Case study

Introduction

In this case study, I will be analyzing fitness tracker data for Bellabeat, a high-tech manufacturer of health-focused products for women.The goal is to gain insights into consumer usage of these products, which will inform Bellabeat’s future marketing strategies.

To address the key business questions, I will follow the steps of the data analysis process: asking questions, preparing data, processing data, analyzing data, sharing findings, and taking action.

Business Task

Urška Sršen , cofounder and Chief Creative Officer of Bellabeat, has requested an analysis of smart device usage data to gain insights into how consumers use non-Bellabeat smart devices. The goal is to apply these insights to a specific Bellabeat product and provide recommendations to influence Bellabeat’s marketing strategy.I’ve chosen to focus my analysis on Leaf, Bellabeat’s classic wellness tracker. This versatile device can be worn as a bracelet, necklace, or clip. The Leaf tracker connects to the Bellabeat app, where it monitors and tracks activity, sleep, and stress levels.

# Prepare data for clustering
activity_data <- dailyActivityUpdated %>%
  group_by(Id) %>%
  summarise(
    avg_steps = mean(TotalSteps, na.rm = TRUE),
    avg_calories = mean(Calories, na.rm = TRUE),
    avg_distance = mean(TotalDistance, na.rm = TRUE)
  )

# Scale the data
scaled_data <- scale(activity_data %>% select(-Id))

# Perform k-means clustering
set.seed(123)
clusters <- kmeans(scaled_data, centers = 3, nstart = 25)

# Add cluster information to the original data
activity_data$cluster <- clusters$cluster

# Print cluster centers
